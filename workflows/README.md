# Design Workflows

This folder contains design workflow documentation shared across all versions.

## Structure

```
workflows/
├── README.md
├── [workflow-name].md      # Individual workflow docs
└── ...
```

## What Goes Here

- Design process workflows (discovery → delivery)
- Handoff procedures (design → dev)
- Review and approval flows
- Component creation workflows
- Design system update processes
- User research workflows
- Prototyping workflows

## Naming Convention

Use kebab-case for workflow files:
- `design-to-dev-handoff.md`
- `component-creation.md`
- `user-research-flow.md`
- `design-review-process.md`

## Template

```markdown
# [Workflow Name]

## Purpose
[What this workflow achieves]

## When to Use
[Trigger conditions]

## Steps
1. [Step 1]
2. [Step 2]
...

## Inputs
- [What's needed to start]

## Outputs
- [What's produced]

## Tools
- [Tools/software used]

## Roles
- [Who's involved]
```
